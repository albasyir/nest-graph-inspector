Use Cases

Why Nest Graph Inspector?

Understand your NestJS architecture at a glance. Designed for teams who need clarity in complex systems.

🎯

Impact Analysis

Narrow regression test scope to the most relevant modules. Understand the blast radius before making changes.

⚡

Test Prioritization

Select critical providers for fast validation. Understand dependency chains and prioritize which flows to check first.

🏗️

Architecture Visibility

Onboard engineers faster, spot highly coupled modules, and make refactors safer by visualizing relationships.

📦

Module Discovery

Automatically discover all loaded modules from the root, including import relationships and provider dependencies.

🔗

Dependency Resolution

Trace internal dependencies, external module dependencies, and NestJS core dependencies grouped under one view.

📄

Multi-Format Output

Export as Markdown with Mermaid graph diagrams or structured JSON — ready for docs, CI pipelines, or custom tools.

Getting Started

Simple Setup

Import the module, set your root module, and you're done. Supports both static and async configuration.

root.module.ts 
import { Module } from '@nestjs/common';
import { AppModule } from './app.module';
import { NestGraphInspector } from 'nest-graph-inspector';

@Module({
  imports: [
    NestGraphInspector.forRoot({
      rootModule: AppModule,
      output: {
        file: 'graph-output.md',
      },
    }),
  ],
})
export class RootModule {}
root.module.ts 
import { Module } from '@nestjs/common';
import { AppModule } from './app.module';
import { NestGraphInspector } from 'nest-graph-inspector';

@Module({
  imports: [
    NestGraphInspector.forRootAsync({
      useFactory() {
        return {
          rootModule: AppModule,
          output: {
            file: 'graph-output.md',
          },
        };
      },
    }),
  ],
})
export class RootModule {}
Reference

Configuration

All available configuration options for NestGraphInspector.

Option Type Description
rootModule Type<any> The root module class to start graph traversal from. This is the entry point of your dependency graph.
output.file string Output file name for the generated Markdown report. The file will include a Mermaid diagram and module details.
💡

The output Markdown file uses Mermaid syntax for graph visualization. Open it in any Mermaid-compatible viewer — GitHub renders it natively, or use the Markdown Preview Mermaid Support extension in VS Code.
