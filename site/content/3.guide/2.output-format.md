## What's Included

Every output — whether Markdown or JSON — contains:

| Section | Description |
| ------- | ----------- |
| **Module list** | All loaded modules from the root |
| **Import relationships** | Which modules import which |
| **Exported providers** | Providers each module exports |
| **Providers with dependencies** | Each provider and what it depends on |
| **Controllers with dependencies** | Each controller and what it depends on |

## Markdown + Mermaid

When outputting to a `.md` file, the graph is rendered as a [Mermaid](https://mermaid.js.org/) diagram embedded in Markdown.

### Viewing Mermaid Diagrams

Mermaid diagrams are supported natively in:

- **GitHub** — renders automatically in `.md` files
- **GitLab** — renders automatically in `.md` files
- **VSCode** — install a Mermaid preview extension

::callout{icon="i-lucide-info" color="primary"}
You can see a real example of the output at [graph-output.md on GitHub](https://github.com/albasyir/nest-graph-inspector/blob/main/src/graph-output.md).
::

## NestJS Core Dependencies

Selected NestJS core dependencies are grouped under a **NestJS Core** module as a Global Module in the graph. This keeps the diagram clean by consolidating framework-internal providers.

## JSON

When outputting to a `.json` file, you get structured data that can be consumed by custom tooling, CI pipelines, or dashboards.
