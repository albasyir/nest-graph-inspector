## rootModule

The `rootModule` option defines the graph entry point. This is the module from which all dependency resolution begins.

```ts
rootModule: AppModule
```

Typically, this is your application's main `AppModule` — the same module you pass to `NestFactory.create()`.

### How it Works

Starting from the root module, Nest Graph Inspector:

1. Reads the module's imports
2. Recursively traverses all imported modules
3. Collects providers, controllers, and their dependencies
4. Builds the complete dependency graph

### Example

```ts [app.module.ts]
import { Module } from '@nestjs/common';
import { UsersModule } from './users/users.module';
import { AuthModule } from './auth/auth.module';

@Module({
  imports: [UsersModule, AuthModule],
})
export class AppModule {}
```

With `rootModule: AppModule`, the graph will include `AppModule`, `UsersModule`, `AuthModule`, and all their transitive dependencies.
